연구 동기
보이스피싱은 음성으로 피해자를 속여 개인정보를 탈취하고 재산상의 이익을 취하는 사기 범죄. AI 기술 활용으로 피해를 크게 줄일 것으로 기대.
주요 기술 동향 (발전 순서)
LSA (Latent Semantic Analysis)
단어-문서 행렬 + SVD로 문서 간 유사성 측정
Word2Vec / Doc2Vec
다층 신경망 기반 단어 임베딩, 단락 간 유사도 계산
BERT
양방향 트랜스포머 인코더 · 16GB 이상 텍스트 사전학습
GPT ← 본 연구
Autoregressive 패러다임 · 대규모 파라미터 및 데이터

1.2 연구 방법 — 제안 기법 개요
"GPT3.5 언어 모델을 활용하여 보이스피싱을 탐지하는 기법 제안"
🤖
제안 기법
GPT text-davinci-003 모델 사용. 한국어 통화 음성 파일을 텍스트로 변환 후 보이스피싱 가능도 답변 → 가능도 기반 탐지 수행.
⚡
핵심 기능
통화 중 주기적 실시간 알람 전송 기능과, 통화 종료 후 최종적으로 보이스피싱 여부를 판단하는 최종 판단 기능 포함.
📊
성능 비교
한국어에 특화된 KoBERT 모델과 비교 분석. 훈련 데이터와 다른 출처의 데이터를 사용하여 범용성 검증.
VP 가능도 정의
GPT가 판단한 Voice Phishing 의심 수준 — 0 ~ 10 사이의 정수로 응답. 0에 가까울수록 일반 대화, 10에 가까울수록 보이스피싱 가능성 高.

전처리 핵심
화자 레이블 변경 — '범죄자'·'피해자' → '참가자1'·'참가자2'로 변경하여 모델이 내용만으로 판단하도록 설계
GPT 처리 절차
Step 1 — 원문 슬라이싱
대화 원문을 Nc개 문자 단위로 분할하여 스텝 생성
Step 2 — 스텝 버퍼 적재
각 스텝을 슬라이딩 버퍼에 입력하여 문맥 유지
Step 3 — GPT Prompting
스텝 버퍼를 포함한 전체 문맥으로 GPT에 VP 가능도 질의
Step 4/5 — 알람 + 최종 판단
실시간 알람 발송 → 통화 종료 후 최종 VP 여부 판정

3.4 VP 가능도 응답 분석 결과
🚨 VP 대화 녹취록
녹취록 1·2·4: VP 가능도 항상 8 이상
녹취록 3·5: 2번을 제외하고 항상 8 이상
녹취록 5번 세부 분석
"긴급한 문제 해결을 위해 돈을 요구하는 내용", "개인정보 유출·범죄 사건 연루 내용" 포함으로 가능도 高. step5 제외 시 가능도 하락.
✅ Non-VP 대화 녹취록
녹취록 3·4·5: VP 가능도 항상 0
녹취록 1·2: 한 번을 제외하고 항상 0
녹취록 1번 세부 분석
"두 사람이 즐겨보는 프로그램에 대해 대화" → 보이스피싱과 무관 판단. 일시적으로 가능도 上승한 step은 "개인정보·범죄 연루 내용" 포함으로 오인.
GPT의 판단 근거
GPT는 개인정보 유출·범죄 사건 연루, 금융 요구, 자신을 검사·경찰이라 주장하는 내용을 보이스피싱 주요 지표로 인식하여 가능도를 부여

Ⅴ. 결론
결론 및 향후 과제
핵심 결과
  • ✅ 정확도 0.95~0.97 달성
  • ✅ 이종 출처 테스트 데이터에서 KoBERT 대비 우수
  • ✅ 텍스트 전처리(tokenization·cleaning) 불필요
GPT 장점 · 단점
장점: 새로운 유형 데이터에도 일관된 성능, 적절한 프롬프트 설계만으로 적용 가능
단점: 토큰당 과금(운용 비용 高), 프롬프트·날짜 변경 시 답변 불안정
향후 연구 과제
  • 📂 더 많은 녹취 데이터 확보 → 성능 개선
  • 🔧 프롬프트 미세 조정 → 정확도·비용 최적화
  • 🏗️ 보이스피싱 전용 중소형 GPT 모델 자체 제작
연구의 의의
보이스피싱 탐지에 GPT 기반 LLM을 최초 적용한 연구. 파인튜닝 없이 Zero-Shot Prompting만으로도 높은 탐지 성능을 입증하여 금융 보안 분야 LLM 응용 가능성 제시.
